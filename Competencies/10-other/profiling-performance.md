# Профилирование и оптимизация производительности

## dotTrace (JetBrains)
- Профилировщик .NET-приложений (Visual Studio extension / standalone).
- **Timeline** — полная картина выполнения (CPU, memory, I/O).
- **Sampling** — быстрый обзор без модификации кода.
- **Tracing** — детальная трассировка по времени (но замедляет).
- **Line-by-line** — профилирование каждой строки кода.

## PerfView
- Бесплатный инструмент от Microsoft для глубокого анализа производительности.
- Работает с ETW (Event Tracing for Windows).
- **GC dump** — анализ сборок мусора.
- **Thread time** — на что тратятся потоки.
- **Memory / JIT / Exception stats**.

## BenchmarkDotNet
- Библиотека для микробенчмарков .NET.
- Предотвращает типичные ошибки замера (jitting, GC warm-up, ввод-вывод).
- Результаты: Mean, Error, StdDev, Rank, Allocated memory.
- `[SimpleJob]`, `[Benchmark]` — простой старт.

## Подходы к оптимизации
1. **Измерь** — без профилирования не оптимизировать (intuition lies).
2. **Узкое место** — найти самый медленный участок.
3. **Правь** — исправить, не нарушая архитектуру.
4. **Проверь** — профилирование после изменений.

## Типичные проблемы
- Много allocations → GC pressure.
- Blocking I/O вместо async.
- String concatenation для больших данных (StringBuilder).
- LINQ с промежуточными allocations.
- Lock contention (много потоков, блокирующих друг друга).

## Важно для интервью
- "Профилирование — единственный способ найти настоящую проблему".
- dotTrace — для высокоуровневого анализа, PerfView — для ETW-глубины.
- BenchmarkDotNet — "для мелких функций".
- Оптимизировать читаемость в первую очередь, производительность — по факту.
