# Value Types vs Reference Types, Boxing/Unboxing, Структуры

## Value Types (типы значений)
- Хранятся **на стеке** (или **inline** внутри reference type).
- При присваивании **копируются** полностью.
- Не могут быть `null` (если не Nullable<T>).
- Бывают:
  - `struct` (пользовательские структуры)
  - `enum` (перечисления)
  - Примитивы (`int`, `double`, `bool`, `char`, `decimal` и т.д.)
- Не поддерживают наследование (но реализуют интерфейсы).

## Reference Types (ссылочные типы)
- Хранятся **в куче (heap)**, на стеке — только ссылка (адрес).
- При присваивании копируется ссылка, а не объект.
- Могут быть `null`.
- `class`, `record class`, `interface`, `delegate`, `string`, `array`, `object`.

## Boxing / Unboxing
- **Boxing** — упаковка value type в `object` (создаётся объект в куче). Дорогая операция:
  1. Выделение памяти в куче
  2. Копирование значения
  3. Потеря информации о типе (но сохраняется через Type)
- **Unboxing** — распаковка обратно. Проверка типа + копирование.
- **Проблемы**: аллокации + дополнительное давление на GC.
- **Избегать**: использовать generics, избегать неявного boxing (например, передачи struct в метод с параметром `object`).

## Структуры (struct)
- Когда использовать: небольшие, иммутабельные, логически единые значения (Point, DateTime, Complex).
- Рекомендации Microsoft: struct должна быть ≤ 16 байт.
- **ref struct** — только на стеке, не может попасть в heap (замыкания, async, boxed). Span<T> — ref struct.
- **readonly struct** — гарантирует иммутабельность, избегает защитного копирования (`in` parameter).
- **record struct** (C# 10) — структурные записи с value equality.

## Record
- **record class** — reference type с value equality, деструктуризацией, `with`-выражениями.
- **record struct** — value type с теми же возможностями (C# 10).

## Важно для интервью
- Чем отличается `struct` от `class`? (хранение, наследование, мутабельность, nullability)
- Когда `struct` выгоднее? (малые размеры, массивы, частые аллокации)
- Проблемы захвата struct в замыкание или async (boxing).
- `in` и `ref` параметры для struct (защитное копирование против `readonly`).
