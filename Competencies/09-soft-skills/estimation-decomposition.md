# Оценка сложности и декомпозиция задач

## Декомпозиция
- Разбиение задачи на более мелкие и понятные части.
- **По вертикали** — слой приложения (UI → API → Service → Repository → DB).
- **По горизонтали** — сквозная функциональность (аутентификация → бизнес-логика → тесты → деплой).
- **Принцип**: каждая подзадача должна быть выполнима за 1-2 дня.
- **User Story splitting** — INVEST: Independent, Negotiable, Valuable, Estimable, Small, Testable.

## Оценка
- **Story Points** (1, 2, 3, 5, 8, 13) — относительная оценка сложности.
- **T-shirt sizes** (S, M, L, XL) — для ранних этапов.
- **Hours/Days** — для детальных задач (у опытных команд может не работать).
- **Affinity estimation** — группировка похожих задач.

## Методы
- **Planning Poker** — команда оценивает, обсуждает расхождения.
- **T-shirt sizing** — быстрая кластеризация.
- **Wideband Delphi** — экспертные оценки с итерациями.
- **Three-point estimation** — Optimistic / Pessimistic / Most Likely.

## Факторы риска
- Неизвестность (технология, домен).
- Зависимости (внешние команды, библиотеки).
- Человеческий фактор (отпуска, больничные).
- Неопределённые требования.

## Важно для интервью
- Относительная оценка (Story Points) точнее абсолютной (часы).
- Velocity — скорость команды в спринте (сумма Point'ов).
- Senior должен защищать команду от нереалистичных сроков (managers).
