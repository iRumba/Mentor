# Проведение технических интервью

## Цели интервью
- Оценить технический уровень кандидата, культурный fit, потенциал.
- Дать кандидату положительный опыт (employer brand).

## Форматы
- **Live coding** — реальная задача на экране (не abacaba — реалистичная задача).
- **System design** — архитектурная задача (спроектировать URL shortener, чат).
- **Soft skills / Behavioural** — STAR-вопросы (Situation, Task, Action, Result).
- **Take-home task** — задание на дом (редко для Senior, трудоёмко для оценки).

## Best Practices
- **Структура** — приветствие, пояснение формата, задача, вопросы кандидату, Q&A.
- **Не молчать** — думать вслух, комментировать, подсказывать при необходимости.
- **Грейдировать** — оценивать не binary (pass/fail), а по уровням.
- **Объективность** — использовать рублику (checklist критериев).
- **Feedback** — собрать мнения всех интервьюверов до принятия решения.

## Ошибки
- **Hiring for culture fit** — найм "похожих на нас" → снижает diversity.
- **Halo effect** — впечатление от одного ответа переносится на всю оценку.
- **Не учитывать nervousness** — кандидат может волноваться.
- **Слишком сложные/простые задачи** — не релевантны для оценки.

## Важно для интервью
- Senior проводит интервью — ожидаемый навык.
- Подготовка — прочитать резюме, подготовить задачи, проверить оборудование.
- Bias awareness — осознавать свои предубеждения.
- Кандидат оценивает компанию так же, как компания оценивает его.
