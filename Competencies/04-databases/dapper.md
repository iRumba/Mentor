# Dapper

## Что такое Dapper
- **Micro-ORM** — минимальная обёртка над ADO.NET на 1 файл.
- Расширение для `IDbConnection` — методы `Query`, `Execute`, `QueryMultiple`.
- Максимальная производительность — почти как raw ADO.NET.

## Преимущества
- **Скорость** — быстрее EF Core в 2–5 раз за счёт отсутствия change tracking.
- **Ручной SQL** — полный контроль над запросами.
- **Лёгкий** — один NuGet-пакет, без громоздкой конфигурации.
- Параметризация — защита от SQL injection.

## Особенности
- Маппинг результата в объекты — по именам колонок (автоматически).
- Multi-mapping — `Query<T1, T2, TReturn>()` для Join.
- `QueryMultiple` — несколько резалт-сетов из одного запроса.
- Dynamic parameters — гибкая передача параметров.

## Важно для интервью
- Когда Dapper vs EF Core: Dapper — если важна производительность или сложные SQL; EF Core — для быстрой разработки.
- Dapper + EF Core вместе: EF Core для обычных CRUD, Dapper для отчётов/сложных запросов.
- Dapper не отслеживает изменения и не генерирует SQL.
- Буферизация vs потоковая обработка (`buffered: true/false`).
