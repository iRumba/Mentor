## SOLID
1. **S**ingle Responsibility — класс имеет одну причину для изменения.
2. **O**pen/Closed — открыт для расширения, закрыт для изменения.
3. **L**iskov Substitution — наследник не должен нарушать контракт базового класса.
4. **I**nterface Segregation — много специализированных интерфейсов лучше одного общего.
5. **D**ependency Inversion — зависит от абстракций, а не конкретных реализаций.
