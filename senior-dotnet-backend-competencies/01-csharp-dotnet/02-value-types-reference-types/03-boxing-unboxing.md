## Boxing / Unboxing
- **Boxing** — упаковка value type в `object` (создаётся объект в куче). Дорогая операция:
  1. Выделение памяти в куче
  2. Копирование значения
  3. Потеря информации о типе (но сохраняется через Type)
- **Unboxing** — распаковка обратно. Проверка типа + копирование.
- **Проблемы**: аллокации + дополнительное давление на GC.
- **Избегать**: использовать generics, избегать неявного boxing (например, передачи struct в метод с параметром `object`).
