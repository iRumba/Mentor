## Важно для интервью

Этот раздел содержит сжатые ответы на самые частые вопросы по теме Middleware и Request Pipeline, которые задают Senior .NET разработчикам.

### 1. `app.Use()` vs `app.Map()` vs `app.Run()`
- **`app.Use()`**: Самый гибкий метод. Позволяет выполнить логику и решить, передавать ли запрос дальше по цепочке (`next`).
- **`app.Map()`**: Используется для ветвления конвейера. Если путь запроса совпадает с указанным, управление передается в отдельную ветку middleware. Запрос из этой ветки обычно не возвращается в основной конвейер.
- **`app.Run()`**: Терминальный middleware. Всегда прерывает цепочку и возвращает ответ.

### 2. Что такое `RequestDelegate` и как он работает?
`RequestDelegate` — это делегат `Func<HttpContext, Task>`. Внутренне ASP.NET Core представляет конвейер как **связный список функций**. Когда вы регистрируете middleware, каждый новый компонент оборачивает предыдущий. В итоге получается одна большая вложенная функция:
`Middleware1(Middleware2(Middleware3(Endpoint)))`.

### 3. Порядок регистрации встроенных middleware (Стандарт)
На интервью часто просят назвать правильный порядок. Ошибка в нем может привести к дырам в безопасности (например, статика доступна без авторизации).

**Рекомендуемый порядок:**
1. `UseExceptionHandler` — чтобы ловить ошибки во всех последующих этапах.
2. `UseHSTS` — безопасность HTTP.
3. `UseHttpsRedirection` — принудительный HTTPS.
4. `UseStaticFiles` — чтобы не тратить ресурсы на Auth/Routing для картинок/css.
5. `UseRouting` — определяет, какой эндпоинт будет вызван.
6. `UseCors` — настраивается на основе данных маршрутизации.
7. `UseAuthentication` — кто этот пользователь?
8. `UseAuthorization` — имеет ли он доступ?
9. `UseEndpoints` (или `MapControllers`) — выполнение бизнес-логики.

### 4. Как реализовать Middleware для логирования времени выполнения запроса?
**Правильный ответ**: Создать Conventional Middleware, где время замеряется до и после `await _next(context)`.

### 5. Можно ли использовать Scoped сервисы в Middleware?
**Ответ**: Да, но **только в методе `InvokeAsync`**. В конструкторе middleware нельзя, так как middleware создается один раз (Singleton) при старте приложения, а Scoped-сервисы создаются на каждый запрос.

### 6. Чем Middleware отличается от Action Filter в ASP.NET Core?
| Признак | Middleware | Action Filter |
| :--- | :--- | :--- |
| **Область** | Весь HTTP-запрос | Конкретный Action в контроллере |
| **Контекст** | Только `HttpContext` | `ActionExecutingContext` (доступ к аргументам метода) |
| **Порядок** | Выполняется раньше всего | Выполняется после Routing и Auth |
| **Цель** | Системные вещи (Логи, Auth, CORS) | Бизнес-валидация, трансформация результата |
