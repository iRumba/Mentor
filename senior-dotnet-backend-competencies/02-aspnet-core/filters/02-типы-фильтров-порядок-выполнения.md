## Типы фильтров (порядок выполнения)

В ASP.NET Core реализовано несколько типов фильтров, которые срабатывают на разных стадиях жизненного цикла запроса к контроллеру.

### 1. Authorization Filters (`IAsyncAuthorizationFilter`)
Выполняются самыми первыми. Их задача — определить, имеет ли пользователь право доступа к данному ресурсу.
- **Особенности**: Если фильтр авторизации возвращает ошибку, остальные фильтры и сам экшен не выполняются.
- **Пример**: Проверка роли пользователя через атрибут `[Authorize]`.

### 2. Resource Filters (`IAsyncResourceFilter`)
Выполняются после фильтров авторизации, но до того, как MVC начнет создавать контроллер (Model Binding).
- **Особенности**: Это "внешний" слой, который оборачивает всё остальное.
- **Кейс использования**: Кэширование ответов. Если ответ в кэше есть, Resource Filter может вернуть его сразу, пропустив создание контроллера и выполнение экшена.

### 3. Action Filters (`IAsyncActionFilter`)
Выполняются непосредственно до и после выполнения метода контроллера.
- **Особенности**: Имеют доступ к `ActionArguments` (аргументам метода) до его вызова и к `ActionResult` после.
- **Кейс использования**: Valdiation, логгирование бизнес-данных, трансформация входящих данных.

### 4. Exception Filters (`IAsyncExceptionFilter`)
Срабатывают только в том случае, если один из предыдущих фильтров или сам экшен выбросил исключение, которое не было перехвачено.
- **Особенности**: Позволяют централизованно обрабатывать ошибки конкретного контроллера или группы методов.
- **Важно**: Они **не** ловят исключения, произошедшие в Middleware.

### 5. Result Filters (`IAsyncResultFilter`)
Выполняются до и после того, как `ActionResult` будет преобразован в реальный HTTP-ответ (например, в JSON или HTML).
- **Особенности**: Позволяют модифицировать результат перед тем, как он будет отправлен клиенту.
- **Кейс использования**: Добавление специфических заголовков в ответ на основе результата выполнения метода.

### Таблица соответствия

| Фильтр | Когда срабатывает | Основная цель |
| :--- | :--- | :--- |
| **Authorization** | 1-й | Доступ |
| **Resource** | 2-й | Оптимизация / Кэш |
| **Action** | 3-й | Бизнес-логика / Валидация |
| **Exception** | При ошибке | Обработка сбоев |
| **Result** | 4-й | Модификация ответа |
