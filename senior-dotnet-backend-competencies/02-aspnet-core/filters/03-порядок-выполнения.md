## Порядок выполнения фильтров

Понимание точного порядка выполнения фильтров необходимо для правильной реализации сквозной логики и отладки поведения API.

### Схема выполнения (Pipeline)

Запрос проходит через фильтры в следующем порядке:

1. **Authorization Filters** $\rightarrow$ (Если OK) $\rightarrow$
2. **Resource Filters (OnResourceExecuting)** $\rightarrow$
3. **Model Binding** $\rightarrow$ (Создание контроллера, заполнение аргументов) $\rightarrow$
4. **Action Filters (OnActionExecuting)** $\rightarrow$
5. **Action Execution** $\rightarrow$ (Выполнение кода в методе контроллера) $\rightarrow$
6. **Action Filters (OnActionExecuted)** $\rightarrow$
7. **Result Execution** $\rightarrow$
    - **Result Filters (OnResultExecuting)** $\rightarrow$
    - **Actual Result Rendering** $\rightarrow$ (JSON $\rightarrow$ HTTP Response) $\rightarrow$
    - **Result Filters (OnResultExecuted)**
8. **Resource Filters (OnResourceExecuted)**

### Особый случай: Exception Filters
Если на любом из этапов (кроме Authorization и первого Resource Filter) происходит исключение, управление передается в **Exception Filters**. 
Если Exception Filter обрабатывает ошибку, выполнение возвращается в цепочку "выхода" (Resource Filter $\rightarrow$ Response).

### Порядок внутри одного типа фильтров (Scope)
Фильтры могут быть зарегистрированы на трех уровнях:
1. **Global** (задаются в `Program.cs`).
2. **Controller** (атрибут на классе контроллера).
3. **Action** (атрибут на конкретном методе).

**Правило обхода:**
- На входе: **Global $\rightarrow$ Controller $\rightarrow$ Action**.
- На выходе: **Action $\rightarrow$ Controller $\rightarrow$ Global**.

### Управление очередью (`Order` property)
Если у вас несколько фильтров одного типа, вы можете управлять их последовательностью с помощью свойства `Order`.
- Чем **меньше** значение `Order`, тем **раньше** выполнится фильтр.
- Значение по умолчанию — 0.

### Пример влияния порядка
Если `FilterA` (Order=1) и `FilterB` (Order=2) являются Action Filters:
- На входе: `FilterA` $\rightarrow$ `FilterB` $\rightarrow$ Action.
- На выходе: `FilterB` $\rightarrow$ `FilterA`.
