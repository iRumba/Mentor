## Что такое фильтры

Фильтры в ASP.NET Core — это механизм, который позволяет выполнять код перед или после выполнения конкретных этапов обработки запроса внутри MVC или API контроллеров.

### Технические детали
В отличие от Middleware, которые работают с `HttpContext` на самом базовом уровне, фильтры имеют доступ к богатому контексту MVC:
- **ActionDescriptor**: Информация о том, какой метод контроллера был вызван.
- **ModelState**: Данные о валидации входящих параметров.
- **ActionArguments**: Значения аргументов, которые будут переданы в метод.

Фильтры позволяют реализовать принцип **Cross-Cutting Concerns** (сквозная функциональность), such as:
- Логирование входящих параметров конкретного метода.
- Валидация бизнес-правил перед вызовом экшена.
- Модификация результата (ActionResult) перед отправкой клиенту.

### Best Practices
- **Используйте async-интерфейсы**: Всегда предпочитайте `IAsyncActionFilter` вместо `IActionFilter` для предотвращения блокировок потоков.
- **Специфичность**: Используйте фильтры там, где вам нужен доступ к данным контроллера/экшена. Если логика нужна для всего приложения (например, CORS), используйте Middleware.
- **Attribute-based**: Для удобства создавайте фильтры в виде атрибутов, чтобы можно было вешать их точечно на методы или контроллеры.

### Сравнение с Middleware
| Характеристика | Middleware | Filter |
| :--- | :--- | :--- |
| **Расположение** | Вне MVC / API | Внутри MVC / API |
| **Доступ к Action** | Нет (только URL) | Полный (метод, аргументы) |
| **Выполнение** | Для каждого запроса | Только для сопоставленных эндпоинтов |
| **Типичный кейс** | Auth, Compression, StaticFiles | Валидация, Кэширование Action |

### Пример: Фильтр для логирования аргументов
```csharp
public class LogArgumentsFilter : IAsyncActionFilter
{
    public async Task OnActionExecutionAsync(ActionExecutingContext context, ActionExecutionDelegate next)
    {
        // Логика ДО выполнения экшена
        var args = string.Join(", ", context.ActionArguments.Select(a => $"{a.Key}={a.Value}"));
        Console.WriteLine($"Вызов метода {context.ActionDescriptor.DisplayName} с аргументами: {args}");

        var resultContext = await next(); // Вызов самого экшена

        // Логика ПОСЛЕ выполнения
        Console.WriteLine($"Метод завершен со статусом {resultContext.Exception?.Message ?? "OK"}");
    }
}
```
