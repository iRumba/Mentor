# Groups (Группы в SignalR)

Группы в SignalR предоставляют механизм для логической организации соединений. Это позволяет отправлять сообщения множеству клиентов, которые объединились по какому-либо признаку, не отслеживая их `ConnectionId` вручную.

## Работа с группами

Управление группами осуществляется через свойство `Groups` внутри Hub.

### Добавление в группу
```csharp
public async Task JoinRoom(string roomName)
{
    await Groups.AddToGroupAsync(Context.ConnectionId, roomName);
    await Clients.Group(roomName).SendAsync("ReceiveMessage", $"User {Context.ConnectionId} joined {roomName}");
}
```

### Удаление из группы
```csharp
public async Task LeaveRoom(string roomName)
{
    await Groups.RemoveFromGroupAsync(Context.ConnectionId, roomName);
}
```

## Технические особенности и ограничения

1. **Отсутствие постоянства (Persistence):** 
   Группы **не сохраняются** в базе данных. Информация о том, кто в какой группе состоит, хранится в памяти сервера (или в Backplane). Если сервер перезагрузится, все членства в группах будут потеряны. Клиенты должны повторно запрашивать вступление в группы при переподключении.
   
2. **Связь с ConnectionId:** 
   В группу добавляется конкретное *соединение* (`ConnectionId`), а не *пользователь* (`UserId`). Если пользователь открыл приложение в трех вкладках браузера, у него будет три разных `ConnectionId`. Чтобы пользователь получал сообщения группы во всех вкладках, вам нужно добавить каждое его соединение в группу.

3. **Автоматическая очистка:** 
   Когда клиент отключается (закрывает вкладку, теряет сеть), SignalR автоматически удаляет его `ConnectionId` из всех групп. Вам не нужно вручную вызывать `RemoveFromGroupAsync` в методе `OnDisconnectedAsync`.

4. **Отсутствие списка участников:** 
   В SignalR **нет встроенного метода** `Groups.GetUsersInGroup("GroupName")`. Это сделано для обеспечения масштабируемости. Если вам нужно знать, кто сейчас находится в комнате, вы должны реализовать этот функционал самостоятельно (например, через Redis или БД).

## Рекомендации по реализации (Best Practices)

- **Интеграция с БД:** Храните информацию о «постоянных» группах (например, подписки на темы) в базе данных. При событии `OnConnectedAsync` проверяйте подписки пользователя и добавляйте его соответствующие `ConnectionId` во все нужные группы SignalR.
- **Именование групп:** Используйте строгие конвенции именования (например, `chat_{roomId}` или `user_notifications_{userId}`), чтобы избежать коллизий.
- **Безопасность:** Никогда не позволяйте клиенту вступать в группу просто по названию, если эта группа требует прав доступа. Всегда проверяйте права пользователя (через `Context.User`) перед вызовом `AddToGroupAsync`.
