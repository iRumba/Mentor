# Основные концепции SignalR

Центральным элементом SignalR является **Hub** — высокоуровневый конвейер, который позволяет клиенту вызывать методы на сервере и наоборот.

## Жизненный цикл Hub

Hub в ASP.NET Core является `transient` по умолчанию. Это означает, что на каждый входящий вызов метода создается новый экземпляр класса Hub. 
**Важно:** Не храните состояние в полях Hub. Для сохранения состояния используйте `HttpContext.Items` (в рамках одного запроса) или распределенный кэш.

## Адресация сообщений (Client Targeting)

SignalR предоставляет мощный API для управления тем, кто получит сообщение:

### 1. Рассылка всем (`Clients.All`)
Отправляет сообщение всем подключенным клиентам, независимо от их группы или пользователя. Мощный инструмент для глобальных системных уведомлений.
```csharp
await Clients.All.SendAsync("ReceiveMessage", "System", "Global maintenance at 22:00");
```

### 2. Рассылка группе (`Clients.Group`)
Позволяет сегментировать пользователей. Например, по ID комнаты чата или по роли (например, "Administrators").
```csharp
await Clients.Group("Room123").SendAsync("ReceiveMessage", "User", "Hello room!");
```

### 3. Рассылка конкретному пользователю (`Clients.User`)
Используется `IUserIdProvider` для сопоставления соединения с идентификатором пользователя (обычно `ClaimTypes.NameIdentifier` из JWT). Один пользователь может иметь несколько активных соединений (разные вкладки браузера), и сообщение придет во все.
```csharp
await Clients.User("userId_abc").SendAsync("ReceiveMessage", "System", "Your order is ready!");
```

### 4. Ответ отправителю (`Clients.Caller`)
Позволяет отправить подтверждение или ошибку только тому клиенту, который инициировал вызов метода.
```csharp
await Clients.Caller.SendAsync("Notify", "Your request was processed");
```

## Strongly-Typed Hubs

Использование строковых имен методов (`"ReceiveMessage"`) чревато ошибками опечаток. Для предотвращения этого используются типизированные хабы:

```csharp
public interface IChatClient {
    Task ReceiveMessage(string user, string message);
}

public class ChatHub : Hub<IChatClient> {
    public async Task SendMessage(string user, string message) {
        await Clients.All.ReceiveMessage(user, message); // Теперь это метод интерфейса
    }
}
```

## Best Practices по разработке

1. **Асинхронность:** Всегда используйте `async/await` (например, `SendAsync`), чтобы не блокировать потоки обработки соединений.
2. **Валидация данных:** Методы Hub — это фактически ваши API-контроллеры. Всегда валидируйте входящие аргументы.
3. **Обработка ошибок:** Ошибки внутри Hub-методов не отправляются клиенту в явном виде по умолчанию. Используйте `try-catch` и отправляйте клиенту специализированное уведомление об ошибке через `Clients.Caller`.
