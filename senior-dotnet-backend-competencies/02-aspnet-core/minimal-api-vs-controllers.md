# Minimal API vs Controllers

## Controllers
- Классы с атрибутами (`[ApiController]`, `[Route]`, `[HttpGet]`).
- Автоматическая валидация модели (`[ApiController]` → 400 без проверки).
- Поддержка фильтров, `ModelState`, привязка модели, Swagger по умолчанию.
- Подходит для сложных проектов с множеством endpoint'ов и единообразной структурой.

## Minimal API
- Функции с лямбда-синтаксисом: `app.MapGet("/items", handler)`.
- Минимальный overhead: нет рефлексии для вызова метода.
- Упрощённая привязка параметров через DI и специальные атрибуты (`[AsParameters]`).
- Явная регистрация всего — больше контроля, меньше магии.

## Сравнение
| Критерий | Controllers | Minimal API |
|---|---|---|
| Сложность | Структурированные | Простые, лаконичные |
| Фильтры | Есть | Через `IEndpointFilter` |
| Swagger | Автоматически | Через `WithOpenApi()` |
| Тестирование | `WebApplicationFactory` | Те же средства |

## Важно для интервью
- Когда выбирать Controllers (большие проекты, единый стиль, фильтры).
- Когда Minimal API (микросервисы, простые CRUD, демо).
- Minimal API появились в .NET 6, значительно расширены в .NET 7-8.
